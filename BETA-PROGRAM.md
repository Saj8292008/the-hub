## Beta Tester Recruitment Criteria

### Primary Profiles

**The Passionate Collector**
- Actively collects watches, sneakers, cars, or similar items
- Regularly monitors multiple marketplaces
- Has missed deals due to timing/lack of awareness
- Values quality over quantity in notifications

**The Savvy Flipper**
- Buys and resells for profit
- Monitors price trends and market opportunities
- Time-sensitive - needs instant alerts
- Understands the value of deal velocity

**The Deal Hunter**
- Loves finding underpriced items
- Active on r/WatchExchange, StockX, Bring a Trailer, etc.
- Has specific search criteria they check manually
- Willing to pay for tools that save time

### Ideal Beta Tester Qualities

- [ ] Active in at least one collecting/flipping community
- [ ] Checks marketplaces daily (or multiple times/day)
- [ ] Comfortable with PWAs and mobile apps
- [ ] Can articulate what makes a "good deal" in their niche
- [ ] Willing to provide honest, detailed feedback
- [ ] Has iOS, Android, or desktop for testing
- [ ] Responsive (can check in weekly)

### Red Flags (Avoid)

- ‚ùå Just wants free stuff with no intention to use it
- ‚ùå Not actually active in any collecting/flipping space
- ‚ùå Can't commit to at least 2 weeks of testing
- ‚ùå Only interested if they get monetary compensation

## Onboarding Flow

### Day 0: Welcome & Setup (30 minutes)

1. **Welcome Email** (automated)
   - Personal thank you
   - Link to beta testing guide
   - Discord/Telegram invite for beta community
   - Link to install PWA

2. **First Login Experience**
   - Quick 3-step onboarding tutorial
   - Set up first "watch" (saved search)
   - Configure notification preferences
   - Set availability hours

3. **Beta Community Intro**
   - Welcome message in beta channel
   - Ice breaker: "What's the best deal you ever found?"
   - Introduce feedback channels

### Week 1: Core Features

- Daily use encouraged
- Test basic notifications
- Set up 2-3 watches in their niche
- First feedback survey (Day 3)

### Week 2: Advanced Features

- Test Pro features (if applicable)
- Notification refinement
- Share deals in community
- Mid-program check-in call (optional, 15 min)

### Week 3-4: Polish & Iteration

- Test bug fixes from earlier feedback
- Help prioritize feature requests
- Prepare for public launch
- Final feedback survey

## Feedback Collection System

### Continuous Feedback Channels

**Discord/Telegram Beta Channel**
- Real-time bug reports
- Feature requests
- Deal sharing and community vibes
- Daily "temperature check" polls

**In-App Feedback Button**
- One-click bug reporting with screenshot
- Feature request voting
- Quick rating prompts after key actions

### Structured Feedback

**Week 1 Survey** (Day 3)
- How easy was setup? (1-5)
- Did you receive your first notification?
- What's confusing or broken?
- What niche are you testing? (watches, sneakers, etc.)

**Mid-Program Interview** (Week 2) - Optional
- 15-minute video/voice call
- Deep dive into use case
- Walk through their workflow
- Identify pain points we're missing

**Week 3 Survey** (Day 21)
- What's working well?
- What's still broken/frustrating?
- Would you pay for this? How much?
- NPS: Would you recommend to a friend? (0-10)

**Final Survey** (End of Week 4)
- Overall experience rating
- Most/least valuable features
- Likelihood to continue using
- Testimonial request (optional)

## Beta Community Guidelines

### Values
- **Honest feedback over politeness** - We want truth, not sugar-coating
- **Specific over vague** - "Notifications are too slow" > "It's not great"
- **Constructive over critical** - Suggest solutions when possible
- **Community over competition** - Help each other, share deals

### Expected Behavior
- ‚úÖ Test regularly (at least 3x/week)
- ‚úÖ Report bugs with details
- ‚úÖ Respond to surveys within 3 days
- ‚úÖ Be respectful to other testers
- ‚úÖ Keep confidential features private until launch
